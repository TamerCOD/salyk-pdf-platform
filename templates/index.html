Как это работает (под капотом)

  1. Excel читается через openpyxl по буквам колонок A..U.
  2. Фаза 1: lookup. Все уникальные ИНН из колонки B прогоняются через cabinet.salyk.kg/TinCheck/GetTaxPayer с паузой 0.25 с. Кэш — один запрос на ИНН, даже если он повторяется в строках.
  3. Фаза 2: render. Для каждой строки: если ИНН нашёлся в ГНС, в колонку C подставляется его наименование (исходное значение C перезаписывается). reportlab рисует overlay-PDF с этими данными в координатах формы.
  4. Фаза 3: group + zip. Опционально склеиваем по выбранной колонке и упаковываем всё в ZIP.
  5. На странице прогресса видны обе фазы с отдельными счётчиками.
// тайминги:
  • Скорость lookup ≈ 0.4–0.6 с / уникальный ИНН.
  • На 100 строк (~50 уникальных ИНН) — ~25 секунд lookup + ~3 секунды render.
  • На 1000 строк (~500 уникальных) — ~3.5 мин lookup + ~30 с render.
  • На 3000 строк (~2000 уникальных) — ~15 мин lookup + ~1.5 мин render.
  • Не закрывайте вкладку без надобности — но даже если закроете, задача продолжится. Адрес страницы прогресса можно сохранить и вернуться к нему.
// поведение для краёв:
  • Если ИНН в строке пустой / меньше 10 цифр — lookup пропускается, в колонку C идёт то, что вы вписали (или пусто).
  • Если ГНС вернул «не найдено» — колонка C остаётся как у вас в Excel.
  • Если ГНС вернул сетевую ошибку — то же самое, плюс на странице прогресса видно, какие ИНН упали красным.
  • Подписи и М.П. на PDF не печатаются — ставятся от руки.
